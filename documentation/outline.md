There are as of now 7 core functionalities that are planned to be implemented.

First is the Read-Eval-Print-Loop (REPL). The goal of the REPL is to parse the users input and tokenize it. This is the foundation of how the user interfaces with the shell.
This is planned to be the longest in terms on time consumption, will have to learn how to use Flex and Bison which are a lexical analyzer generator and GNU parser generator respectively.

Second is Builtins. The built in commands we want for basic functionality. Examples include help, cd, pwd, etc.

Once these are established, more technical features can be added such as fork-exec-wait, pipes, file redirection, signal handling, and lastly AI functionality.
For these features the functionality of the shell developed in Operating Systems Programming will be useful and hopefully will speed up the development.
